# Pipeline Architecture

Detailed research flow for both pipeline modes. For project overview see [README.md](../README.md); for system architecture see [architecture.md](architecture.md).

**Pipelines:**
- [Scorecard Pipeline](#scorecard-pipeline) — evaluates a research question across weighted dimensions
- [Matrix Pipeline](#matrix-pipeline) — compares multiple subjects across multiple attributes

---

## Scorecard Pipeline

Evaluates a research question across weighted dimensions. Each dimension receives an independent score, confidence level, evidence trail, and structured arguments.

```mermaid
flowchart TD
    INPUT["🔬 User Input\n════════════════════════════════════════════\nResearch description (free-text query)\nDimensions with weights and researchHints\nResearch setup: decisionContext, userRoleContext\nEvidence mode: native │ deep-assist"]

    QS["① Query Strategist\n════════════════════════════════════════════\nActor: Analyst LLM (retrieval capability)\nModel (default): gemini-2.5-flash\nRequest: 1 LLM call, no web search\n────────────────────────────────────────────\n▶ Sent: description, dimension list with labels\n  and researchHints.queryTemplates/whereToLook\n──────────── ▼ Response ────────────\n◀ Returns JSON:\n  • niche: short domain label\n  • aliases: company/product rebrand hints\n  • dimensionHints per dimension:\n    - querySeeds: 2-4 supporting search queries\n    - counterfactualQueries: 2-3 disconfirming queries (RQ-05)\n    - sourceTargets: preferred source types"]

    DA_CHECK{"Evidence mode?"}

    P1_NATIVE["② Native Phase 1 — Baseline + Web + Reconcile\n════════════════════════════════════════════\nActor: Analyst LLM\nModel (default): baseline/reconcile/scoring = gpt-5.4-mini\nModel (default): web evidence pass = gemini-2.5-flash\nRequests: 3 structured passes\n────────────────────────────────────────────\nPass A: baseline (memory-only)\nPass B: web-assisted (live web search)\nPass C: reconcile baseline+web (no web search)\n────────────────────────────────────────────\nOutput (per dimension):\n  • score: 1-5 numeric\n  • confidence: high │ medium │ low\n  • confidenceReason: why this confidence level\n  • brief: 1-2 sentence summary\n  • full: detailed evidence paragraph\n  • sources[]: {name, quote, url, sourceType}\n  • arguments: {supporting[], limiting[]}\n    each: {id, claim, detail, sources[]}\n  • risks: free-text risk description\n  • missingEvidence: what's still unknown\n◀ Also returns:\n  • attributes: {title, sector, geography}\n  • conclusion: overall assessment"]

    DA_PARALLEL["②᛫ Deep Research ×3 — Collect\n════════════════════════════════════════════\nActors: ChatGPT (gpt-5.4) + Claude (claude-sonnet-4)\n  + Gemini (gemini-2.5-pro)\nRequests: 3 LLM calls in parallel, each with web search\n────────────────────────────────────────────\n▶ Sent to each: same prompt as Phase 1 — description,\n  dimensions, strategist hints, research setup\n──────────── ▼ Response ────────────\n◀ Each returns: complete scorecard payload\n  (same schema as Phase 1 output)\n◀ Per-provider meta tracked separately:\n  • liveSearchUsed, webSearchCalls\n  • durationMs, providerId"]

    DA_MERGE["②᛫ Deep Research ×3 — Merge\n════════════════════════════════════════════\nNo LLM call — deterministic logic\nFunction: mergeDeepAssistScorecardPayloads()\n────────────────────────────────────────────\nPer dimension:\n  • Pick provider with highest confidence\n  • Average scores across all successful providers\n  • Merge source lists, deduplicate by name│quote│url\n  • Merge arguments (supporting + limiting)\n  • Compute providerAgreement:\n    agree │ partial │ contradict\nTop-level:\n  • Merge attributes from best provider\n  • Combine conclusion from highest-confidence provider"]

    DA_RECOVER["②᛫ Deep Research ×3 — DA-02 Recovery\n════════════════════════════════════════════\nTrigger: providerAgreement=contradict, confidence=low,\n  or sourceCount < 2 after merge\nSelection: selectDeepAssistRecoveryDimensions()\n  ranks by pressure score (contradiction +5, low conf +4,\n  sparse sources +2-3, provider failure +1)\nBudget: limits.deepAssistRecoveryBudget (default: up to 4)\nModel (default): query/search = gemini-2.5-flash\nModel (default): rescore = gpt-5.4-mini\n────────────────────────────────────────────\nPer weak dimension (sequential, 3 LLM calls each):\n  ▶ Query Plan: sent current dim state + strategist hints\n  ◀ returns {gap, queries[3-4], counterfactualQueries[2-3]}\n  ──────────── ▼ ────────────\n  ▶ Search Harvest: sent queries + dim context\n  ◀ returns {findings[]: {query, evidenceType, fact, source}}\n    each finding labeled supporting │ counterfactual\n  ──────────── ▼ ────────────\n  ▶ Rescore: sent original dim + harvest findings\n  ◀ returns updated {score, confidence, arguments, sources}\n    counterfactual findings → limiting arguments"]

    TARGETED["③ Targeted Recovery\n════════════════════════════════════════════\nTrigger: dimensions with confidence=low or\n  medium + specific missingEvidence gap\nSelection: selectTargetedCycleDimensions()\n  ranks by: confidence level, source count,\n  missing evidence specificity, stale ratio\nBudget: limits.targetedBudgetUnits (default: up to 4)\nModel (default): query/search = gemini-2.5-flash\nModel (default): rescore = gpt-5.4-mini\n────────────────────────────────────────────\nPer weak dimension (sequential, 3 LLM calls each):\n  ▶ Query Plan: dim state, strategist seeds, research hints\n  ◀ returns {gap, queries[3-4], counterfactualQueries[2-3]}\n  ──────────── ▼ ────────────\n  ▶ Search Harvest: queries + current dim snapshot\n  ◀ returns {findings[], queryCoverage[]}\n  ──────────── ▼ ────────────\n  ▶ Rescore: original dim + harvest + query plan\n  ◀ returns updated score/confidence/arguments/sources\n────────────────────────────────────────────\nRefinement: if zero dimensions upgraded on first pass,\n  entire cycle repeats once on same candidates"]

    SV1["④ Source Verification\n════════════════════════════════════════════\nNo LLM calls — HTTP fetches via transport.fetchSource()\n────────────────────────────────────────────\nPer source URL in every dimension:\n  1. Fetch page content (12s timeout)\n  2. fuzzyQuoteCoverage(): check if quote appears\n  3. quotedClaimFoundInPage(): semantic match\n  4. Assign verificationStatus:\n     • verified_in_page — quote found\n     • name_only_in_page — name found, no quote\n     • not_found_in_page — nothing matched\n     • fetch_failed — HTTP error\n     • invalid_url — malformed\n  5. Derive displayStatus (UX-02):\n     • cited — verified_in_page\n     • corroborating — name_only + non-vendor\n     • excluded_marketing — vendor + not verified\n     • excluded_stale — ≥60% stale + only old years\n     • unverified — everything else\n────────────────────────────────────────────\nPenalties applied:\n  • applySourceVerificationPenalty(): reduce confidence\n    if too many sources fail verification\n  • applyEvidenceQualityCaps(): cap confidence based on\n    stale evidence ratio, vendor source ratio,\n    zero-source dimensions"]

    CRITIC["⑤ Critic Audit\n════════════════════════════════════════════\nActor: Critic LLM\nModel (default): claude-sonnet-4-20250514\nRequest: 1 LLM call, live web search enabled\nToken budget: ~6K\n────────────────────────────────────────────\n▶ Sent: complete Phase 1 output — all dimension\n  scores, evidence, arguments, sources,\n  confidence levels, research setup context\n──────────── ▼ Response ────────────\n◀ Returns JSON per dimension:\n  • agreedScore: critic's independent score\n  • proposedScore: what critic thinks it should be\n  • flag: true if critic disagrees\n  • flagReason: explanation of disagreement\n  • flagSources[]: critic's own sources as evidence\n────────────────────────────────────────────\nCritic's flagSources also go through\nsource verification (same as step ④)"]

    RESPONSE["⑥ Analyst Response (Phase 3)\n════════════════════════════════════════════\nActor: Analyst LLM, live web search enabled\nModel (default): gpt-5.4-mini\nRequest: 1 LLM call\nToken budget: ~6K\n────────────────────────────────────────────\n▶ Sent: Phase 1 scores + all critic flags +\n  critic sources + critic proposed scores\n──────────── ▼ Response ────────────\n◀ Returns JSON per dimension:\n  • score: final reconciled score\n  • confidence: updated confidence level\n  • brief/full: updated evidence text\n  • sources[]: merged analyst + critic sources\n  • arguments: updated supporting + limiting\n  • risks: updated risk text\n  • acceptedFlag: whether critic flag was accepted"]

    CONSISTENCY["⑦ Consistency & Coherence\n════════════════════════════════════════════\nRequests: 2 LLM calls\nModel (default): consistency=analyst gpt-5.4-mini (live)\nModel (default): coherence=critic claude-sonnet-4-20250514 (live)\n────────────────────────────────────────────\nConsistency check:\n  ▶ Sent: all dimension scores + confidence levels\n  ◀ Returns: flags where score ordering is illogical\n──────────── ▼ ────────────\nCoherence audit:\n  ▶ Sent: all dimension evidence + conclusion\n  ◀ Returns: contradictions across dimensions\n  and narrative inconsistencies"]

    SV2["⑧ Final Source Verification\n════════════════════════════════════════════\nSame process as step ④\nRe-verifies any new sources introduced\nduring critic/response/consistency phases"]

    REDTEAM["⑨ Red Team (RQ-02)\n════════════════════════════════════════════\nActor: Critic LLM, no web search\nModel (default): claude-sonnet-4-20250514\nRequest: 1 LLM call\n────────────────────────────────────────────\n▶ Sent: final scores, conclusion,\n  per-dimension: score, confidence, risks,\n  top 2 limiting arguments, missingEvidence,\n  providerAgreement\n──────────── ▼ Response ────────────\n◀ Returns JSON:\n  • redTeamVerdict: 1-2 sentence counter-case\n  • per dimension:\n    - threat: strongest counter-argument\n    - missedRisk: risk not already explicit\n    - severityIfWrong: high │ medium │ low\n────────────────────────────────────────────\nDoes NOT change scores — only appends risk context\nvia applyScorecardRedTeam() into dimension.risks"]

    SYNTH["⑩ Synthesizer (RQ-09)\n════════════════════════════════════════════\nActor: Synthesizer LLM (falls back to Critic model)\nModel (default fallback): claude-sonnet-4-20250514\nRequest: 1 LLM call, no web search\n────────────────────────────────────────────\n▶ Sent: structured signals ONLY, no raw source prose:\n  • weightedScore, conclusion\n  • providerContributions (which models contributed)\n  • redTeam verdict and findings\n  • per dimension: score, confidence, confidenceReason,\n    limiting arguments, risks, providerAgreement\n──────────── ▼ Response ────────────\n◀ Returns JSON:\n  • executiveSummary: 3-5 sentence synthesis\n  • decisionImplication: what this means right now\n  • keyUncertainties[]: top evidence gaps\n  • dissent: strongest case against conclusion\n────────────────────────────────────────────\ndecisionImplication replaces analyst's original\nconclusion in the final output"]

    DISCOVER["⑪ Discovery\n════════════════════════════════════════════\nActor: Analyst LLM\nModel (default): gpt-5.4-mini\nRequest: 1 LLM call, live web search enabled\n────────────────────────────────────────────\n▶ Sent: completed scorecard summary,\n  dimension scores and evidence gaps\n──────────── ▼ Response ────────────\n◀ Returns: suggested follow-up research threads\n  with rationale for each"]

    OUTPUT["📊 Final Scorecard Output\n════════════════════════════════════════════\nPer dimension:\n  • finalScore (1-5), confidence (high/med/low)\n  • brief, full evidence text\n  • sources[] with displayStatus + verificationStatus\n  • arguments: {supporting[], limiting[]}\n  • risks (analyst + red team appended)\n  • missingEvidence, confidenceReason\nTop-level:\n  • weightedScore, conclusion (from synthesizer)\n  • executiveSummary: {executiveSummary,\n    decisionImplication, keyUncertainties, dissent}\n  • redTeam: {redTeamVerdict, dimensions}\n  • sourceUniverse: {cited, corroborating,\n    unverified, excludedMarketing, excludedStale}\n  • analysisMeta: full run diagnostics\n  • discovery suggestions"]

    INPUT --> QS --> DA_CHECK
    DA_CHECK -->|"Verified Research (native)"| P1_NATIVE --> TARGETED
    DA_CHECK -->|"Deep Research ×3"| DA_PARALLEL --> DA_MERGE --> DA_RECOVER --> TARGETED
    TARGETED --> SV1 --> CRITIC --> RESPONSE --> CONSISTENCY --> SV2 --> REDTEAM --> SYNTH --> DISCOVER --> OUTPUT
```

## Matrix Pipeline

Compares multiple subjects across multiple attributes. Each cell (subject × attribute) receives independent evidence, confidence, and structured arguments.

```mermaid
flowchart TD
    INPUT["🔬 User Input\n════════════════════════════════════════════\nResearch description (free-text query)\nDecision question (explicit or inferred)\nSubjects[] (or auto-discover)\nAttributes[] with briefs and optional derived flags\nResearch setup: decisionContext, userRoleContext\nEvidence mode: native │ deep-assist"]

    SD["① Subject Discovery (optional)\n════════════════════════════════════════════\nActor: Analyst LLM\nModel (default): gpt-5.4-mini\nRequest: 1 LLM call, live web search enabled\nTrigger: explicit+extracted subjects are below minCount\n────────────────────────────────────────────\n▶ Sent: research description, decision question,\n  subjectsSpec: {minCount, maxCount} + setup context\n──────────── ▼ Response ────────────\n◀ Returns JSON:\n  • suggestedSubjects[]: {label, reason, sources[]}\n  (e.g., competitor products to compare)"]

    BASE["② Analyst Baseline — Populate Grid (Memory-Only)\n════════════════════════════════════════════\nActor: Analyst LLM\nModel (default): gpt-5.4-mini\nRequest: chunked pass, live web search disabled\n────────────────────────────────────────────\nOutput: per-cell value/full/confidence/sources/arguments"]

    WEB["③ Analyst Web Pass — Populate Grid\n════════════════════════════════════════════\nActor: Analyst LLM with live web search\nModel (default): gpt-5.4-mini\nRequest: chunked pass, live web search enabled\nToken budget: ~14K (web)\n────────────────────────────────────────────\n▶ Sent: description, decision question,\n  subjects[], attributes[], research setup context\n──────────── ▼ Response ────────────\n◀ Returns JSON — per cell (subject × attribute):\n  • value: concise assessment text\n  • full: detailed evidence paragraph\n  • confidence: high │ medium │ low\n  • confidenceReason: explanation\n  • sources[]: {name, quote, url, sourceType}\n  • arguments: {supporting[], limiting[]}\n    each: {id, claim, detail, sources[], group, status}\n  • risks: free-text risk description"]

    RECON["④ Reconcile Baseline + Web\n════════════════════════════════════════════\nActor: Analyst LLM\nModel (default): gpt-5.4-mini\nRequest: chunked pass, live web search disabled\n────────────────────────────────────────────\nMerges baseline/web drafts and applies reconcile\nquality guard + retry when merge is implausible"]

    TARGETED["⑤ Targeted Recovery\n════════════════════════════════════════════\nTrigger: cells with confidence=low after reconcile\nSelection: selectMatrixTargetedCells()\n  ranks by: matrixCellPressure() — confidence,\n  source count, stale ratio, evidence gaps\nBudget: adaptive limits.matrixTargetedBudgetCells\nModel (current runtime): query/search/rescore = gpt-5.4-mini\n────────────────────────────────────────────\nPer weak cell (sequential, 3 LLM calls each):\n  ▶ Query Plan (no web search)\n  ▶ Search Harvest (live web search)\n  ▶ Rescore (no web search)\ncounterfactual findings → limiting arguments"]

    DA_CHECK{"Evidence mode?"}

    DA_PARALLEL["⑤᛫ Deep Research ×3 — Enrichment Collect\n════════════════════════════════════════════\nActors: ChatGPT (gpt-5.4) + Claude (claude-sonnet-4)\n  + Gemini (gemini-2.5-pro)\nRequests: 3 LLM calls in parallel, each with web search\n────────────────────────────────────────────\n▶ Sent to each: same matrix prompt — description,\n  decision question, subjects, attributes, research setup\n──────────── ▼ Response ────────────\n◀ Each returns: complete matrix grid\n  (same per-cell schema as analyst pass)\n◀ Per-provider meta tracked:\n  • liveSearchUsed, webSearchCalls, durationMs"]

    DA_MERGE["⑤᛫ Deep Research ×3 — Merge & Reconcile\n════════════════════════════════════════════\nNo LLM call — deterministic logic\nFunction: mergeDeepAssistMatrix()\n────────────────────────────────────────────\nPer cell:\n  • Pick provider with highest confidence\n  • matrixProviderAgreement(): token overlap scoring\n    between provider responses for same cell\n    ≥0.42 = agree, ≥0.22 = partial, else contradict\n  • Merge source lists, deduplicate\n  • Merge arguments (supporting + limiting)\n  • Assign providerAgreement label per cell"]

    DA_RECOVER["⑤᛫ Deep Research ×3 — DA-02 Recovery\n════════════════════════════════════════════\nTrigger: providerAgreement=contradict, confidence=low,\n  or sourceCount < 2 after merge\nSelection: selectDeepAssistRecoveryCells()\n  ranks by pressure (contradiction +5, low conf +4,\n  sparse sources +2-3, provider failure +1)\nBudget: limits.deepAssistRecoveryBudgetCells (default: up to 6)\n────────────────────────────────────────────\nPer weak cell (sequential, 3 LLM calls each):\n  ▶ Query Plan: sent cell state + strategist hints\n  ◀ returns {gap, queries[3-4], counterfactualQueries[2-3]}\n  ──────────── ▼ ────────────\n  ▶ Search Harvest: sent queries + cell context\n  ◀ returns {findings[]: {query, evidenceType, fact, source}}\n  ──────────── ▼ ────────────\n  ▶ Rescore: sent original cell + harvest\n  ◀ returns updated {value, confidence, arguments, sources}\n────────────────────────────────────────────\nSources re-verified after all recovery cells complete"]

    SV["⑥ Cell Source Verification\n════════════════════════════════════════════\nNo LLM calls — HTTP fetches via transport.fetchSource()\n────────────────────────────────────────────\nPer source URL in every cell:\n  1. Fetch page content (12s timeout)\n  2. Check quote presence (fuzzy matching)\n  3. Assign verificationStatus:\n     verified_in_page │ name_only │ not_found │\n     fetch_failed │ invalid_url\n  4. Derive displayStatus (UX-02):\n     cited │ corroborating │ excluded_marketing │\n     excluded_stale │ unverified\n  5. Apply per-cell confidence penalties\n  6. Quality caps for stale/vendor evidence\n────────────────────────────────────────────\nRuns after each source-producing phase\n(baseline/web/reconcile/targeted/deep-assist,\ncritic-response, derived attributes)"]

    EARLY["⑦ Early Catastrophic Coverage Gate\n════════════════════════════════════════════\nNo LLM call — deterministic guardrail\n────────────────────────────────────────────\nAborts before critic when post-targeted coverage\nis catastrophically low across subjects/cells"]

    CRITIC["⑧ Critic Audit\n════════════════════════════════════════════\nActor: Critic LLM\nModel (default): claude-sonnet-4-20250514\nRequest: 1 LLM call, live web search enabled\nToken budget: ~6K\n────────────────────────────────────────────\n▶ Sent: complete matrix grid — all cells with\n  values, confidence, sources, arguments,\n  decision question, research setup context\n──────────── ▼ Response ────────────\n◀ Returns JSON per cell:\n  • agreedValue: critic's independent assessment\n  • proposedValue: what critic thinks it should be\n  • flag: true if critic disagrees\n  • flagReason: explanation of disagreement\n  • flagSources[]: critic's own evidence"]

    RESPONSE["⑨ Analyst Response\n════════════════════════════════════════════\nActor: Analyst LLM, live web search enabled\nModel (default): gpt-5.4-mini\nRequest: 1 LLM call\n────────────────────────────────────────────\n▶ Sent: matrix grid + all critic flags +\n  critic sources + proposed values\n──────────── ▼ Response ────────────\n◀ Returns JSON per flagged cell:\n  • value: reconciled assessment\n  • confidence: updated level\n  • arguments: updated supporting + limiting\n  • acceptedFlag: whether critic flag accepted\n  • justification: why accepted or rejected"]

    CONSISTENCY["⑩ Consistency Audit\n════════════════════════════════════════════\nActor: Critic LLM\nModel (default): claude-sonnet-4-20250514\nRequest: 1 LLM call, no web search\n────────────────────────────────────────────\n▶ Sent: all cells grouped by subject,\n  cross-subject score patterns\n──────────── ▼ Response ────────────\n◀ Returns: flags where comparable cells across\n  subjects have illogical score relationships\n  (e.g., Subject A clearly stronger on attribute X\n  but scored lower than Subject B)"]

    DERIVED["⑪ Derived Attributes\n════════════════════════════════════════════\nActor: Analyst LLM\nModel (default): gpt-5.4-mini\nRequest: 1 LLM call for derived-attribute set\n────────────────────────────────────────────\n▶ Sent: completed matrix cells that this\n  derived attribute set depends on\n──────────── ▼ Response ────────────\n◀ Returns: computed column values\n  (e.g., composite scores, rankings)\n────────────────────────────────────────────\nOnly generated after all evidence, critic,\nand consistency steps are complete"]

    REDTEAM["⑫ Red Team\n════════════════════════════════════════════\nActor: Critic LLM, no web search\nModel (default): claude-sonnet-4-20250514\nRequest: 1 LLM call\n────────────────────────────────────────────\n▶ Sent: matrix state — executiveSummary,\n  crossMatrixSummary, up to 24 cells prioritized\n  by lowest confidence + provider contradictions\n  Per cell: value, confidence, risks, providerAgreement\n──────────── ▼ Response ────────────\n◀ Returns JSON:\n  • redTeamVerdict: 1-2 sentence counter-case\n  • per cell:\n    - threat: strongest counter-argument\n    - missedRisk: risk not explicit in current cell\n    - severityIfWrong: high │ medium │ low\n────────────────────────────────────────────\nDoes NOT change values — appends risk context\nvia applyMatrixRedTeam() into cell.risks"]

    SYNTH["⑬ Synthesizer\n════════════════════════════════════════════\nActor: Synthesizer LLM (falls back to Critic model)\nModel (default fallback): claude-sonnet-4-20250514\nRequest: 1 LLM call, no web search\n────────────────────────────────────────────\n▶ Sent: structured signals ONLY, no raw source prose:\n  • coverage stats, providerContributions\n  • redTeam verdict and cell findings\n  • crossMatrixSummary\n  • per cell: confidence, confidenceReason, value,\n    risks, providerAgreement, top 2 limiting args\n──────────── ▼ Response ────────────\n◀ Returns JSON:\n  • decisionAnswer: direct answer to decision question\n  • closestThreats: nearest competitive threats\n  • whitespace: main opportunity identified\n  • strategicClassification: concept classification\n  • keyRisks: top viability risks\n  • decisionImplications: recommended next steps\n  • uncertaintyNotes: major evidence gaps\n  • providerAgreementHighlights: where providers\n    agreed or disagreed"]

    SLA["⑭ Coverage + Decision-Grade Gates\n════════════════════════════════════════════\nNo LLM call — deterministic validation\n────────────────────────────────────────────\nChecks against limits.matrixCoverageSLA and\nlimits.matrixDecisionGradeGate:\n  • minSourcesPerCell: minimum sources each cell needs\n  • minSubjectEvidenceCoverage: fraction of cells\n    that must have adequate evidence\n  • maxUnresolvedCellsRatio: max fraction of cells\n    still at low confidence\nFailing gates mark run as degraded\n(or abort in strict quality mode)"]

    DISCOVER["⑮ Discovery\n════════════════════════════════════════════\nActor: Analyst LLM\nModel (default): gpt-5.4-mini\nRequest: 1 LLM call, live web search enabled\n────────────────────────────────────────────\n▶ Sent: completed matrix summary,\n  subjects, attributes, evidence gaps\n──────────── ▼ Response ────────────\n◀ Returns JSON:\n  • suggestedSubjects[]: {label, reason}\n  • suggestedAttributes[]: {label, reason}\n  • notes: optional observations"]

    OUTPUT["📊 Final Matrix Output\n════════════════════════════════════════════\nPer cell (subject × attribute):\n  • value, full evidence text\n  • confidence + confidenceReason\n  • sources[] with displayStatus + verificationStatus\n  • arguments: {supporting[], limiting[]}\n  • risks (analyst + red team appended)\n  • providerAgreement (if Deep Research ×3)\nTop-level:\n  • executiveSummary: {decisionAnswer, closestThreats,\n    whitespace, keyRisks, decisionImplications, ...}\n  • redTeam: {redTeamVerdict, cells}\n  • crossMatrixSummary, coverage stats\n  • sourceUniverse: {cited, corroborating,\n    unverified, excludedMarketing, excludedStale}\n  • analysisMeta: full run diagnostics\n  • discovery: {suggestedSubjects, suggestedAttributes}"]

    INPUT --> SD --> BASE --> WEB --> RECON --> TARGETED --> DA_CHECK
    DA_CHECK -->|"Verified Research (native)"| SV
    DA_CHECK -->|"Deep Research ×3"| DA_PARALLEL --> DA_MERGE --> DA_RECOVER --> SV
    SV --> EARLY --> CRITIC --> RESPONSE --> CONSISTENCY --> DERIVED --> REDTEAM --> SYNTH --> SLA --> DISCOVER --> OUTPUT
```
