## Open Items

Baseline: ~18 API calls per scorecard analysis run; ~3 calls per matrix analysis run.

| ID | Type | Problem | Solution | Impact | Extra calls | Cost +% | Priority |
|----|------|---------|----------|--------|-------------|---------|----------|
| RQ-01 (Fact-Checker) | feature | Analyst cites "Company X achieved Y% ROI" — nobody verifies whether the claim is real. Critic shares the same training data and hallucinates the same way. Both models confirm each other's fabrications. | Add a **Fact-Checker** phase between Analyst reconcile and Critic. For each dimension, extract top 3-5 named claims (company, metric, deployment), run targeted web searches to verify each. Output per claim: `verified / partially_verified / unverified / contradicted` with source. Feed results into Critic and Analyst Response so unverified claims lose confidence. Merge RA-14 into this track to avoid duplicate verification logic. | Catches hallucinated claims — the single biggest accuracy gap shared by all LLM research tools. | +5-15 | +25-80% | P1 |
| RA-10 | improvement | Critic reviews each dimension independently and misses cross-dimension contradictions (for example high evidence + very low ROI). | Add a cross-dimension coherence check block in critic prompt with explicit contradiction rules to flag. | Fewer structurally inconsistent final scorecards. | 0 | 0% | P1 |
| RA-11 | improvement | Polarity for `regulatory`, `change_mgmt`, `build_vs_buy` is still occasionally inverted in model outputs. | Add explicit "do NOT invert polarity" warnings in Phase 1 scoring and Phase 3 prompts for those dimensions. | Fewer wrong-direction score updates. | 0 | 0% | P1 |
| RA-12 | improvement | Weighted score ignores confidence; low-confidence 4/5 contributes equally to high-confidence 4/5. | Add confidence-weighted score alongside raw score (discounts: high 1.0, medium 0.85, low 0.65) and show both in UI. | Better ranking trust under uncertainty. | 0 | 0% | P2 |
| RQ-02 (Red Team) | feature | Critic audits existing claims but doesn't construct the case *against* the thesis. It asks "is this score justified?" — not "what would make this fail?" | Add a **Red Team** pass parallel with Critic. For each dimension, search for: failed implementations, incumbents that make the approach unnecessary, regulatory/organizational barriers. Produces negative evidence the analyst must address in Phase 4. Consider implementing together with RQ-05 as one disconfirming-evidence stream. | Surfaces what the analyst *didn't* say. No other research tool actively tries to disprove its own findings. | +3-6 | +15-30% | P2 |
| RQ-03 (Multi-Model Ensemble) | feature | Analyst and Critic are both GPT — same training data, same blind spots. The "debate" is one brain arguing with itself. | Add Claude provider adapter (`engine/providers/anthropic.js`) and Gemini provider adapter (`engine/providers/google.js`). Extend `ResearchConfig.models` to support per-phase provider. Recommended: GPT = Analyst, Gemini = web-search phases, Claude = Critic + Reconciler. Extend transport to route by phase. | Cross-model adversarial review catches ~30-50% of hallucinated claims that same-model debate misses. | ~same | ~0% | P3 |
| RQ-05 (Counterfactual Evidence) | feature | All search queries are framed to *support* the dimension. Nobody searches for "why AI fails in insurance claims." | For each dimension, generate 2-3 counterfactual queries (failure cases, barriers, competitor advantages). Feed results into reconcile as a third evidence stream alongside baseline and web. Consider implementing together with RQ-02 as one disconfirming-evidence track. | Balances evidence — no other tool actively searches for disconfirming evidence. | +3-6 | +15-30% | P2 |
| CF-02 | improvement | New config dimensions (all configs except engine defaults) have no `polarityHint` or `researchHints`. Low-confidence cycle uses generic fallback queries. | Add `polarityHint` and `researchHints` (`whereToLook`, `queryTemplates`) to all dimensions in all config definitions. | More targeted re-search queries for every config. | 0 | 0% | P1 |
| MX-04 | improvement | No cross-subject consistency check. If Competitor A claims "best pricing" but Competitor B's cell shows lower prices, contradiction is not caught. | Add cross-subject consistency audit after critic response. Compare cells within same attribute across all subjects, flag contradictions, reconcile. | Catches the matrix-specific failure mode: comparisons that contradict each other. | +1 | +33% of matrix baseline | P1 |
| MX-05 | improvement | Derived attributes (`derived: true`, e.g. "Editorial Priority", "Verdict") are generated in the same pass as evidence attributes, before evidence is finalized. | Sequence derived attributes after all evidence cells are finalized (post-critic, post-consistency). Separate analyst call receives complete evidence matrix and produces only derived cells. | Derived judgments grounded in full critic-reviewed evidence, not analyst's first draft. | +1 | +33% of matrix baseline | P1 |
| DM-01 (Demo Research Pack) | feature | New users land in an empty workspace and cannot quickly see output quality/shape across scorecard and matrix modes. | Add a static demo pack loaded in UI by default (read-only or tagged as demo). | RQ-06 (Temporal Evidence Gating) | improvement | Sources can be 3+ years old. Analyst cites a 2022 case study as current evidence for a 2026 decision.
