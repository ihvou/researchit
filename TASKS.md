## Open Items — Native Pipeline

Baseline (current observed range): ~18 API calls per scorecard run; ~10-20 calls per matrix run when low-confidence recovery is active.

Priority scale: P0 = reliability blocker / correctness bug, P1 = strong quality lift, P2 = strategic lift, P3 = deep/expensive optional mode.

Classic-flow target stack (quality-first default routing): OpenAI reasoning model as Analyst, Anthropic model as Critic, Gemini-grounded retrieval/query planning for web evidence, with fallback mesh when one provider degrades.

## Deep Assist — see `docs/spec-deep-assist.md`

Deep Assist introduces an alternative evidence collection layer that uses external deep research APIs (Claude, ChatGPT, Gemini) instead of internal web_search-based retrieval. The critic, scoring, verification, and synthesis layers remain shared with the Native pipeline.

User selects evidence mode at research launch time — not in config. Same config works with either mode. Deep Assist default is 3 providers (quality-first) with no hard spend cap, but with strict retry/timeout safeguards and full cost transparency.
